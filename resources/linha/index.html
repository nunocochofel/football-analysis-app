Sobre este protótipo

Este protótipo roda inteiramente no navegador e cobre o fluxo completo — importar, marcar, desenhar, editar e extrair estatísticas. Algumas partes profissionais dependem de infraestrutura dedicada (servidor, GPU) e aqui estão representadas de forma simplificada e honesta:

  • Rastreamento por IA de jogadores/bola — não incluído aqui; exigiria um modelo de visão computacional rodando em GPU dedicada. A ferramenta "Seguir jogador" (🎯) faz um rastreamento manual: pausa o vídeo automaticamente, marcas a posição do alvo, avanças um pouco (pausado) e marcas outra vez — o desenho interpola entre os pontos. Podes seguir vários alvos ao mesmo tempo (incluindo a bola), rever/apagar pontos individuais na lista, e ver a distância entre alvos ativos.
  • Apagar desenhos — botão direito em qualquer desenho remove-o; a ferramenta 🧹 apaga por clique/arrasto.
  • Captura via câmeras físicas/RTSP em tempo real — não é possível num app de navegador; o modo "Sem vídeo" simula a codificação ao vivo na beira do campo com cronômetro.
  • Exportação de clips — usa a API nativa de gravação do navegador (MediaRecorder), gerando ficheiros .webm reais (vídeo + desenhos + zoom, se aplicado) — não .mp4, porque gerar MP4 verdadeiro dentro de um navegador exige uma biblioteca pesada (ffmpeg compilado para WebAssembly) que este protótipo não inclui. A exportação corre em tempo real: um clip de 10s demora ~10s a exportar. Se precisares de .mp4, converte o .webm depois (ex.: HandBrake). Em Chrome/Edge escolhes onde guardar — um clip individual abre "Guardar como", e "Exportar tudo" pede uma pasta uma única vez e guarda todos os clipes lá dentro; noutros browsers cai para a transferência automática de cada ficheiro.
  • Modelagem 3D de estádio e câmera virtual — o quadro tático aqui é 2D, com objetos (bola/cone/baliza) e cenas salvas que são reproduzidas em sequência com transição animada entre posições.
  • Zoom no vídeo — desliza o controlo 🔍 na barra de ferramentas; com a ferramenta ✋ ativa, arrasta para escolher a zona ampliada (com outra ferramenta ativa, mantém Shift para arrastar). Com um clip selecionado para edição, o zoom aplicado fica gravado nesse clip e é usado na exportação.
  • Régua de distância — funciona de verdade após calibrar a escala (desenhe uma linha de referência e informe o comprimento real).
  • Arquivos grandes — o carregamento usa referência direta ao arquivo (sem copiar tudo para a memória), então arquivos de vários GB são suportados em teoria. Na prática, o limite real é o que o navegador consegue decodificar, não o tamanho do arquivo. Se travar, o app avisa depois de alguns segundos em vez de ficar com tela preta indefinidamente.
  • Formatos de vídeo — navegadores só decodificam os codecs/contêineres que trazem embutidos. MP4 (H.264 + AAC) é o mais compatível universalmente; .mov com HEVC/ProRes, .mkv, .avi e .wmv costumam falhar no Chrome/Firefox mesmo sendo arquivos válidos — não há solução via código para isso dentro de um app de navegador, é preciso converter o arquivo antes (ex.: HandBrake, ffmpeg) para MP4/WebM.
  • Atalhos de teclado — teclas já mapeadas marcam o evento na hora; qualquer tecla de letra/número ainda sem categoria abre um criador rápido para você inventar novos tipos de corte sem abrir o menu de configurações.
  • Esc cancela um rastreamento ou zona em desenho.

Cada projeto (times, marcações, cenas) fica guardado separadamente no navegador. Usa "← Projetos" no topo para voltar à lista e abrir/criar outro — abre em duas abas do browser para trabalhar em dois projetos ao mesmo tempo.
